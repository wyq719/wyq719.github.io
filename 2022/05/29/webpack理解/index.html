Webpack理解？解决什么问题

2022-05-29
字数统计: 762   |   阅读时长≈ 2 分钟

Webpack最初的目标是是西安前端项目的模块化，旨在更高效的管理和维护项目中的每一个资源

一、背景

模块化

最早的时候，我们通过文件划分的形式实现模块化，就是将每个功能以及相关的数据各自单独放到不同的js文件中。每一个文件就是一个单独的模块，然后再将这些js文件一如到页面，一个script标签对应一个模块，然后再调用模块化的成员

<script src="module-a.js"></script>
<script src="module-b.js"></script>

随后出现了命名空间方式，每个模块只能暴露一个全局对象，然后模块的内容都挂载到这个对象中。

但这种方式也没有解决依赖等问题，再后来我们使用立即执行函数模块作为提供私有空间，通过参数的形式作为依赖声明。但是这种方式维护起来非常麻烦

理想的解决方式是，在页面中引入js入口文件，其余模块可以通过代码控制按需加载，还需要规定模块化的规范，比如common AMD CMD ES Modules

缺点：模块都是在全局中工作，大量模块成员污染了环境，模块与模块之间并没有依赖关系、维护困难、没有私有空间等问题

二、问题

现代前端开发已经变得十分的复杂，所以我们开发过程中会遇到如下的问题：

  • 需要通过模块化的方式来开发
  • 使用一些高级的特性来加快我们的开发效率或者安全性，比如通过ES6+、TypeScript开发脚本逻辑，通过sass、less等方式来编写css样式代码
  • 监听文件的变化来并且反映到浏览器上，提高开发的效率
  • JavaScript 代码需要模块化，HTML 和 CSS 这些资源文件也会面临需要被模块化的问题
  • 开发完成后我们还需要将代码进行压缩、合并以及其他相关的优化

而webpack恰巧可以解决以上问题

三、是什么

webpack 是一个用于现代JavaScript应用程序的静态模块打包工具，在web开发过程中打包ht js css等静态资源，开发更加高效。对于不同类型的资源，webpack有模块加载器对模块包分析他们之间的依赖关系，最终优化合并为静态资源。

Webpack的能力：

编译代码能力，提高效率，解决浏览器兼容问题

模块整合能力，提高性能，可维护性，解决浏览器频繁请求文件的问题

万物皆可模块能力，项目维护性增强，支持不同种类的前端模块类型，统一的模块化方案，所有资源文件的加载都可以通过代码控制
